Getting Started

From zero to checking out branches with PR status in under five minutes. This guide walks you through installation, first run, and GitHub integration.

Prerequisites

Before installing git-switchboard, make sure you have the following in place.

  • git 2.20 or newer installed and on your PATH
  • A terminal emulator with UTF-8 support (virtually all modern terminals qualify)
  • For GitHub integration: a GitHub account and a personal access token (created during setup)
Info

git-switchboard ships as a self-contained binary. There are no runtime dependencies — no Node.js, no Python, no interpreters required.

Installation

Choose the installation method that matches your platform and package manager preference.

Homebrew (macOS / Linux)

Shell
RECOMMENDED
$ brew tap craigory/git-switchboard
$ brew install git-switchboard

Chocolatey (Windows)

PowerShell (Admin) 
PS> choco install git-switchboard

Install Script (macOS / Linux)

Shell 
$ curl -fsSL https://git-switchboard.dev/install.sh | sh

npm / Bun / pnpm

Shell 
$ npm install -g git-switchboard
# or:
$ bun add -g git-switchboard

Verify the installation by running:

Shell
$ git switchboard --version
git-switchboard v0.3.0

First Run

Navigate to any git repository and run the branch picker:

Shell
$ cd ~/repos/my-project
$ git switchboard

The TUI will launch immediately, showing all local branches sorted by most-recently-updated.

Navigating Branches

The branch picker is keyboard-driven. Use the arrow keys or vim-style j/k to move through the list. Press ↵ to check out the highlighted branch.

Fuzzy Search

Press / to activate search mode. Start typing any part of the branch name, author, or PR title. Results narrow instantly. Press Esc to clear and return to the full list.

Filtering by Author

Press a to toggle between showing only your own branches and showing all authors. The current filter is displayed in the status bar at the top of the TUI.

Tip

Set ui.default_author in your config to automatically filter to your branches on every launch — useful on shared repos with many contributors.

GitHub Integration Setup

To see PR numbers, CI status, and review states in the branch picker, you need a GitHub personal access token.

Step 1: Create a Token

Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic). Create a token with the repo scope.

Warning

Copy your token immediately — GitHub only shows it once. Store it in a password manager.

Step 2: Add to Config

~/.config/git-switchboard/config.toml
[github]
token = "ghp_yourTokenHere"

Or set it as an environment variable:

Shell
$ export GITHUB_TOKEN=ghp_yourTokenHere

Step 3: Verify

Relaunch git switchboard. The PR column should now populate with linked PR numbers and states.

PR Dashboard

Once GitHub integration is active, launch the PR dashboard to see all open pull requests across your repositories:

Shell
$ git switchboard pr

From the PR dashboard you can open PRs in your browser (↵), clone the branch locally (c), or open it in your configured editor (e). Press R to retry CI on the selected PR.

Configuration

The full config file lives at ~/.config/git-switchboard/config.toml. A minimal working configuration looks like this:

config.toml
[github]
token = "ghp_..."

[ui]
show_remote = false
default_author = ""

See the full Configuration Reference for all available options.
